# 기능 목록

## 게임 상태 관리

- [ ] 초기 상태 생성 (체력 100, 식량 3, 감염도 10, Day 1)
- [ ] 치료 횟수(healAttempts), 구조 포인트(rescuePoints) 추적
- [ ] 카드 덱 생성 및 셔플

## 카드 덱 시스템

- [ ] 20장 카드 덱 생성 (6종류, 분배표에 따라)
- [ ] Fisher-Yates 셔플 알고리즘
- [ ] 카드 뽑기 (덱에서 1장 pop)
- [ ] 덱 소진 시 자동 리셔플

## 카드 종류 및 선택지

- [ ] 생존자 시체 (4장): 배낭째로 가져온다 (식량 +3, 감염 +8) / 겉에 있는 것만 집는다 (식량 +1)
- [ ] 부상당한 군인 (4장): 식량을 건네고 약을 받는다 (식량 -1, 감염 -20, 치료 +1) / 몸싸움 끝에 식량만 챙기고 떠난다 (체력 -10, 식량 +2)
- [ ] 임시 수술 (3장): 감염 부위를 직접 도려낸다 (체력 -25, 감염 -25, 치료 +1) / 이를 악물고 참는다 (체력 -5, 감염 +10)
- [ ] 군용 차량 행렬 (3장): 뛰어나가 신호를 보낸다 (구조 +1, 감염 +8) / 몸을 낮추고 방향만 확인한다 (체력 +5)
- [ ] 오염된 웅덩이 (3장): 그냥 마신다. 탈수보다 낫다 (체력 +5, 감염 +15) / 참는다. 빗물을 기다린다 (체력 -10)
- [ ] 구조 트럭 (3장): 전력으로 달려간다 (체력 -20, 구조 +1) / 체력을 아끼고 쉰다 (체력 +10)

## 선택지 처리

- [ ] 선택 후 2초(2000ms) 대기
- [ ] 카드 효과 먼저 적용 → 이후 일일 효과(식량 -1, 감염 +3) 적용
- [ ] 치료 선택 시 healAttempts 증가
- [ ] 구조 응답 시 rescuePoints 증가

## 일일 자동 효과

- [ ] 식량 1 소비 (최솟값 0)
- [ ] 감염도 3 증가 (최솟값 0)
- [ ] 식량이 0인 상태에서 선택 시 체력 10 추가 감소

## 엔딩 조건

- [ ] 사망: 체력 <= 0
- [ ] 좀비화: 감염도 >= 100
- [ ] 치료 성공: 치료 선택 5회 누적
- [ ] 구조 성공: 구조 포인트 3 이상 + Day > 10
- [ ] 생존 성공: Day > 15
- [ ] 포기

## 엔딩 결과 출력 (화면 표시)

- [ ] 생존 일수, 최종 체력, 최종 식량, 최종 감염도, 엔딩 표시

## UI

- [ ] 현재 Day 및 스탯 표시 (체력, 식량, 감염도, 치료 횟수, 구조 포인트)
- [ ] 남은 카드 수 표시
- [ ] 카드 뽑기 버튼
- [ ] 카드 이름/설명 표시
- [ ] 선택지 A/B 버튼
- [ ] 선택 후 2초 대기 (로딩 표시)
- [ ] 게임 로그 표시
- [ ] 게임 종료 화면
- [ ] 다시 시작 버튼
- [ ] 포기 버튼

## 입력 유효성 검사

- [ ] 선택 처리 중 버튼 비활성화
- [ ] 게임 종료 후 입력 차단
