# 좀비 아포칼립스: 감염자의 이지선다 - 기능 목록

## 기능 목록

### 1. 게임 초기화

- 플레이어 초기 상태: 체력 100, 식량 3, 감염도 10
- Day 1, 치료 횟수 0, 구조 포인트 0 표시
- 카드 뽑기 버튼 활성화
- 선택지 영역, 결과 화면 숨김
- 20장 카드 덱 생성 및 셔플

### 2. 카드 덱 구성

- 20장의 카드로 구성된 덱을 셔플하여 사용
- 생존자 시체 (4장)
  - A: 배낭째로 가져온다 (식량 +3, 감염도 +8)
  - B: 겉에 있는 것만 집는다 (식량 +1)
- 부상당한 군인 (4장)
  - A: 식량을 건네고 약을 받는다 (식량 -1, 감염도 -20)
  - B: 몸싸움 끝에 식량만 챙기고 떠난다 (체력 -10, 식량 +2)
- 임시 수술 (3장)
  - A: 감염 부위를 직접 도려낸다 (체력 -25, 감염도 -25)
  - B: 이를 악물고 참는다 (체력 -5, 감염도 +10)
- 군용 차량 행렬 (3장)
  - A: 뛰어나가 신호를 보낸다 (구조 +1, 감염도 +8)
  - B: 몸을 낮추고 방향만 확인한다 (체력 +5)
- 오염된 웅덩이 (3장)
  - A: 그냥 마신다 (체력 +5, 감염도 +15)
  - B: 참는다. 빗물을 기다린다 (체력 -10)
- 구조 트럭 (3장)
  - A: 전력으로 달려간다 (체력 -20, 구조 +1)
  - B: 체력을 아끼고 쉰다 (체력 +10)
- 덱이 소진되면 자동으로 리셔플

### 3. 카드 뽑기

- 카드 뽑기 버튼 클릭 시 덱에서 1장 뽑기
- 남은 카드 수 업데이트
- 카드 이름, 설명, 선택지 A/B 표시
- 카드 뽑기 버튼 숨김, 선택지 영역 표시

### 4. 선택지 선택 및 효과 적용

- 선택지 클릭 시 버튼 비활성화, 로딩 표시
- 2초(2000ms) 후 결과 적용:
  1. 카드 효과 먼저 적용
  2. 식량 1 소비
  3. 감염도 3 증가
  4. 식량이 0인 상태에서 선택 시 체력 10 추가 감소
- 스탯 최솟값: 체력 0, 식량 0, 감염도 0

### 5. 일차 진행

- 선택 완료 후 Day 1 증가
- 스탯 실시간 업데이트 (치료 횟수, 구조 포인트 포함)
- 카드 뽑기 버튼 재표시, 선택지 영역 숨김

### 6. 엔딩 판정

- 여러 엔딩 조건이 동시에 충족될 경우, 실패 엔딩(사망 / 좀비화)을 성공 엔딩보다 우선 적용한다.
- 실패 엔딩 (성공보다 우선)
  - 체력 <= 0 -> 사망
  - 감염도 >= 100 -> 좀비화
- 성공 엔딩
  - 치료 선택 5회 누적 -> 치료 성공 (부상당한 군인 A + 임시 수술 A)
  - 구조 포인트 >= 3 + Day > 10 -> 구조 성공
  - Day > 15 -> 생존 성공

### 7. 게임 종료 및 결과 화면

- 포기 버튼 클릭 시 즉시 종료
- 결과: 생존 일수, 최종 체력, 최종 식량, 최종 감염도, 엔딩 표시

### 8. 다시 시작

- 다시 시작 버튼 클릭 시 초기 상태로 리셋

## 코딩 컨벤션

- var 사용 금지, const와 let 사용
- 전역 변수 사용 금지
- 외부 라이브러리 사용 금지, 순수 Vanilla JS만 사용
- Airbnb 자바스크립트 코드 컨벤션 준수
- indent depth 2 이하
- 함수(메소드)는 한 가지 일만 하도록 최대한 작게
- 함수(메소드) 길이 15라인 이하
- import 문으로 모듈화
