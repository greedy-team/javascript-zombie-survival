## 🎯 기능 요구사항

원인 불명의 감염 사태가 발생했다. 도시는 봉쇄되었고, 당신은 **이미 좀비에게 물렸다.**
완전히 좀비로 변하기 전에 **치료제를 완성**하거나, **15일을 버텨 구조대를 기다려야** 한다.

이 게임은 **웹 브라우저에서 실행**된다. 플레이어는 버튼을 클릭해 카드를 뽑고 선택하며, 모든 결과는 화면에 표시된다.

- 게임 시작 시 플레이어의 초기 상태는 **체력 100, 식량 3, 감염도 10, Day 1**이며, 치료 횟수와 구조 포인트는 **0**이다.
- 매일(Day) **카드 1장을 뽑고**, 카드의 **선택지 2개 중 1개를 고른다.**
- 카드를 뽑으면 뽑기 버튼이 사라지고 카드와 선택지가 화면에 표시된다.
- 선택지를 고르면 버튼이 비활성화되고 로딩이 표시된다. **2초(2000ms)** 후 결과가 반영되며, 다음 카드를 뽑을 수 있는 상태로 돌아온다.
- 매일 선택지를 고르면 아래 순서로 처리된다:
  1. 카드 효과가 적용된다
  2. **현재 식량이 0인지 확인한다** (기아 판정)
  3. 식량 1이 소비된다 (0 아래로 내려가지 않는다)
  4. 감염도가 3 증가한다
  5. 2번에서 식량이 0이었다면, **체력이 10 추가 감소**한다

  > 예시: 초기 상태(체력 100, 식량 3, 감염도 10)에서 '생존자 시체' A(식량 +3, 감염도 +8) 선택 시
  > → 식량: 3 + 3 - 1(기본 소비) = **5**, 감염도: 10 + 8 + 3(기본 증가) = **21**

- 치료 선택과 구조 응답 횟수가 **화면 상단 스탯에 실시간으로 표시**된다.
- 체력은 **0 미만으로 내려가지 않는다.** (최솟값 0)
- 식량은 **0 미만으로 내려가지 않는다.** (최솟값 0)
- 감염도는 **0 미만으로 내려가지 않는다.** (최솟값 0, 0은 일시적 억제 상태로 완치가 아님)
- 플레이어는 언제든 **게임을 포기**할 수 있다.
- 게임이 종료되면 결과 화면으로 전환되며, **재시작 버튼**이 표시된다. 재시작 시 모든 상태와 로그가 초기화된다.

### 🃏 카드 덱 규칙

20장의 카드로 구성된 덱을 셔플하여 사용한다. 매일 **1장을 뽑고 선택지를 고른다.**
덱이 소진되면 자동으로 리셔플된다. 현재 덱에 남은 카드 수는 화면에 실시간으로 표시된다.

| 카드              | 장수 | 선택지 A 효과                     | 선택지 B 효과       |
| ----------------- | ---- | --------------------------------- | ------------------- |
| 🧟 생존자 시체    | 4    | 식량 +3, 감염도 +8                | 식량 +1             |
| 💊 부상당한 군인  | 4    | 식량 -1, 감염도 -20 ✅ 치료       | 체력 -10, 식량 +2   |
| 🔪 임시 수술      | 3    | 체력 -25, 감염도 -25 ✅ 치료      | 체력 -5, 감염도 +10 |
| 🚗 군용 차량 행렬 | 3    | 구조 포인트 +1, 감염도 +8 🚁 구조 | 체력 +5             |
| 💧 오염된 웅덩이  | 3    | 체력 +5, 감염도 +15               | 체력 -10            |
| 🚛 구조 트럭      | 3    | 체력 -20, 구조 포인트 +1 🚁 구조  | 체력 +10            |

각 카드의 **이름**과 **선택지별 효과 수치**는 반드시 위 표의 값과 일치해야 한다. 카드에 표시할 설명 텍스트와 선택지 버튼의 문구는 카드 이름과 효과를 참고하여 자유롭게 작성하면 된다.

### 🏁 엔딩 조건

#### 실패 엔딩

- **체력 ≤ 0** → 사망
- **감염도 ≥ 100** → 좀비화

#### 성공 엔딩

- **치료 선택 5회 누적** → 치료 성공
- **구조 포인트 3 이상 + Day > 10** → 구조 성공
- **Day > 15** → 생존 성공 (구조대 도착)

---

#### ⚠️ 엔딩 우선순위

여러 엔딩 조건이 동시에 충족될 경우, 실패 엔딩(사망 / 좀비화)을 성공 엔딩보다 우선 적용한다.

### 📋 게임 로그

게임 진행 상황은 화면의 로그 영역(`#log`)에 실시간으로 기록된다.

- 다음 시점에 로그를 기록해야 한다:
  - 게임 시작 시
  - 카드를 뽑았을 때
  - 선택지를 골랐을 때
  - 기아가 발생했을 때
- 로그 메시지의 텍스트는 **자유롭게 작성**할 수 있다
- 단, **기아 발생 시 로그에는 반드시 `식량이 없어`라는 문구가 포함**되어야 한다 (테스트 검증 대상)

## ✅ 프로그래밍 요구사항

- `var`를 사용하지 않는다. `const`, `let`만 사용한다.
- 전역 변수를 사용하지 않는다.
- 외부 라이브러리를 사용하지 않고, 순수 Vanilla JS로만 구현한다.
- **Airbnb 자바스크립트 코드 컨벤션**을 지키면서 프로그래밍 한다.
- indent(인덴트, 들여쓰기) depth를 2가 넘지 않도록 구현한다.
  - 예를 들어 while문 안에 if문이 있으면 들여쓰기는 2이다.
  - 힌트: indent depth를 줄이는 좋은 방법은 함수(또는 메소드)를 분리하는 것이다.
- 함수(또는 메소드)가 한 가지 일만 하도록 최대한 작게 만들어라.
- `import` 문을 이용해 스크립트를 모듈화하고 불러올 수 있게 만든다.
- 함수(또는 메소드)의 길이가 15라인을 넘어가지 않도록 구현한다.

## ✔️ 테스트 실행 가이드

- 테스트 실행에 필요한 패키지 설치를 위해 `Node.js` 버전 `20` 이상이 필요하다.
- 다음 명령어를 입력해 패키지를 설치한다.

```
npm install
```

- 설치가 완료되었다면, VSCode의 **Go Live** 버튼을 클릭해 로컬 서버(`http://localhost:5500`)를 먼저 실행한다.

- 서버가 실행된 상태에서 다음 명령어를 입력해 테스트를 실행한다.

```
npm run test
```

- 모든 테스트가 pass한다면 성공!
